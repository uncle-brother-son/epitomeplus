# Development Standards for Epitome Plus

## Project Context
- Next.js 16+ with App Router and React 19+
- Sanity CMS headless backend (project: e3cginxl)
- Tailwind CSS v4 for styling
- Deployment: Cloudflare Workers + Pages
- Target: Production portfolio site with high performance standards

## Critical Requirements

### Content Freshness (TOP PRIORITY)
- **Content updates in Sanity MUST be visible within seconds**
- Use `dynamic='force-dynamic'` and `revalidate=0` on all pages
- Sanity CDN (60-90s cache) is acceptable
- ISR with hours/days revalidation is NOT acceptable
- Service workers must not interfere with instant content updates

### Performance Targets
- LCP: < 2.5s
- INP: < 200ms  
- TTFB: < 0.6s
- Mobile bandwidth optimization is critical
- Core Web Vitals "Good" ratings required

## Code Quality Standards

### React Best Practices

#### Component Optimization
- **Always use `React.memo()` on pure/stateless components**
  ```tsx
  export const MyComponent = memo(function MyComponent({ prop }: Props) {
    // component logic
  });
  ```
- Wrap with memo: Schema components, display-only components, cards/lists
- Don't wrap: Components with heavy context dependencies, frequently changing props

#### Hooks & Performance
- Use `useMemo()` for expensive computations
- Use `useCallback()` for event handlers passed to memoized children
- Memoize debounced/throttled functions to prevent recreation:
  ```tsx
  const debouncedFn = useMemo(() => debounce(handler, 150), []);
  ```

### Event Handlers

#### High-Frequency Events (CRITICAL)
- **Scroll**: Always throttle (16ms for 60fps) + `{ passive: true }`
  ```tsx
  const throttledScroll = throttle(handleScroll, 16);
  window.addEventListener('scroll', throttledScroll, { passive: true });
  ```
- **Resize**: Always debounce (150ms minimum)
  ```tsx
  const debouncedResize = debounce(handleResize, 150);
  window.addEventListener('resize', debouncedResize);
  ```
- **Touch/Mouse Move**: Throttle at 16-32ms depending on use case
- All scroll/touch listeners must use `{ passive: true }` flag

#### Utility Functions Location
- Use `/web/src/app/lib/utils.ts` for debounce/throttle utilities
- Import and apply to all high-frequency event handlers

### Images & Media

#### Next.js Image Component
- **Always** use Sanity image URL builder from `@/lib/sanity/image`
- Default quality: `quality={75}` (balance of size vs quality)
- Always include `placeholder="empty"` to prevent layout shift
- Responsive sizes attribute required:
  ```tsx
  sizes="(max-width: 767px) 100vw, (max-width: 1024px) 50vw, 800px"
  ```
- First visible images: `loading="eager"` or `priority`
- Off-screen images: `loading="lazy"`

#### Video Optimization
- Autoplay videos: `preload="metadata"` (not "auto")
- User-controlled videos: `preload="metadata"`
- Always include: `muted`, `loop`, `playsInline` for autoplay
- Never use `preload="auto"` unless explicitly required

#### Carousel/Grid Patterns
- First 3 items: `loading="eager"`
- Remaining items: `loading="lazy"`
- Use Sanity URL builder width based on container size

### Intersection Observer

#### Animation Components
- Always include `rootMargin: '50px'` to trigger before viewport entry
- Prevents flash of pre-animation state
- Standard config:
  ```tsx
  new IntersectionObserver(callback, { 
    threshold: 0.1, 
    rootMargin: '50px' 
  });
  ```

### Next.js Specific

#### Data Fetching
- **Parallelize independent queries** with `Promise.all()`:
  ```tsx
  const [nav, sitedata, footer] = await Promise.all([
    getNavigation(),
    getSitedata(),
    getFooter(),
  ]);
  ```
- Never fetch sequentially when queries are independent

#### Sanity Integration
- Sanity client must have `useCdn: true` (60-90s edge caching)
- Use projection in queries to fetch only needed fields
- Optimize image URLs server-side with width/quality params

#### Link Prefetching
- Critical navigation (main nav): Default prefetch behavior
- Secondary links (footer, info pages): `prefetch={false}`
- Balance instant navigation vs bandwidth usage

#### External Resources
- Preconnect to critical domains in layout.tsx:
  ```tsx
  <link rel="preconnect" href="https://cdn.sanity.io" />
  <link rel="preconnect" href="https://fonts.googleapis.com" />
  <link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="" />
  ```

#### Scripts & Analytics
- Google Analytics: Use `strategy="lazyOnload"`
- Third-party scripts: Lazy load when possible
- Defer non-critical scripts

### TypeScript

#### Type Safety
- Always define proper TypeScript types/interfaces
- Export types from query files for reuse
- Use `PortableTextBlock` type for Sanity rich text fields
- Avoid `any` - use proper generics

#### Component Props
- Define interfaces for all component props
- Use destructuring with defaults:
  ```tsx
  function Component({ items = [], loading = 'lazy' }: Props) {}
  ```

### Page Transitions

#### Animation Timing
- Maintain 640ms transition duration (brand aesthetic preference)
- Use MutationObserver to wait for content readiness
- Include 3-second safety timeout
- Smooth ease-in-out timing functions

### Browser Compatibility
- All animations: `ease-epitome` custom timing
- Stick to widely supported features
- Test on mobile Safari and Chrome

## Anti-Patterns (DO NOT DO)

### Performance
- ❌ Console.log statements in production code
- ❌ Inline arrow functions in JSX props (creates new reference each render)
- ❌ Event listeners without debounce/throttle on high-frequency events
- ❌ Missing `passive: true` on scroll/touch listeners
- ❌ `preload="auto"` on videos
- ❌ IntersectionObserver without `rootMargin`

### Next.js
- ❌ Sequential data fetching when queries are independent
- ❌ Aggressive ISR caching (conflicts with instant update requirement)
- ❌ Service workers that cache content (conflicts with instant updates)
- ❌ Missing dynamic config on pages that need fresh data

### React
- ❌ Creating functions inside render without useCallback
- ❌ Not memoizing pure components
- ❌ Expensive computations without useMemo
- ❌ Default exports without memo wrapper on pure components

### Images/Media
- ❌ Using Next Image without responsive sizes
- ❌ Missing placeholder attribute
- ❌ Not using Sanity URL builder (wastes bandwidth)
- ❌ Same loading strategy for all images

## File Structure Conventions

### Component Files
- Client components: `'use client'` directive at top
- Server components: No directive (default)
- One component per file (except small related helpers)
- Co-locate types with components when component-specific

### Naming
- Components: PascalCase (ProjectCard.tsx)
- Utilities: camelCase (utils.ts)
- Queries: get* prefix (getFooter.ts)
- Types: Match domain (FooterData, NavItem)

## Testing Priorities

### Manual Testing Required
- Scroll performance on mobile devices
- Image loading on slow 3G
- Animation smoothness at 60fps
- Sanity content updates appear within seconds after publish
- Page transitions feel deliberate (640ms timing maintained)

### Performance Monitoring
- Check DevTools Performance tab for:
  - Long tasks blocking main thread
  - Excessive event listener firing
  - Layout shifts during load
  - Memory leaks from event listeners

## Brand/Design Requirements
- Page transitions should feel deliberate, not rushed
- 640ms transition timing preferred over faster alternatives
- Animations should complete smoothly without jank
- Mobile experience is equally important as desktop

## When in Doubt
1. Optimize for performance first
2. Respect the instant update requirement
3. Use React.memo() on pure components
4. Debounce/throttle event handlers
5. Ask before introducing aggressive caching strategies
